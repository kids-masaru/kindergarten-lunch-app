# 機能仕様書：日次注文確認・配送計画ビュー

作成日: 2026-03-27
ステータス: 計画中

---

## 背景・目的

「電話で"あそこ何食だっけ？"と聞かれたときにパッと見てすぐわかる画面がほしい」
「今日の配送ルートを組む際に、どの幼稚園でどのエリアかが一覧できるといい」

現状の管理画面には以下のギャップがある：
- 注文確認は月単位のカレンダー印刷がベース → 特定日を即確認しにくい
- 配送計画に必要な「エリア情報」がデータに存在しない
- 上記2つをカバーする専用ビューがない

---

## 対象ユーザー

管理者（ママミレ運営スタッフ）

---

## 機能概要

管理画面に新セクション **「今日の注文」** を追加する。
このセクションは2つのビューをタブで切り替える。

| タブ | 名称 | 用途 |
|---|---|---|
| ① | 注文一覧 | 指定日の全園の注文を即確認 |
| ② | 配送リスト | 同日の配送計画をエリア別に整理 |

---

## 前提：住所・エリアフィールドの追加

配送計画に必要なため、幼稚園マスターに `address`（住所）と `area`（エリア）フィールドを新設する。

### エリア自動抽出の仕組み

住所フィールドへの入力をトリガーに、JavaScriptが自動でエリアを抽出する。

**抽出優先順位：区 → 市 → 町/村**

| 入力住所例 | 自動抽出されるエリア |
|---|---|
| 大阪府大阪市北区梅田1-2-3 | 北区 |
| 東京都新宿区西新宿2-8-1 | 新宿区 |
| 神奈川県横浜市港北区日吉4-1-1 | 港北区 |
| 北海道函館市本町12-3 | 函館市（区なし → 市） |
| 長野県松本市大手1-2-3 | 松本市 |

エリアは自動入力後も手動で修正可能（自動抽出がうまくいかない場合のフォールバック）。

### 変更対象

| 対象 | 変更内容 |
|---|---|
| Googleスプレッドシート `kindergartens` シート | `address`・`area` カラムを自動追加（auto-create 機構を活用） |
| `backend/models.py` | `KindergartenMaster` に `address: str = ""`・`area: str = ""` を追加 |
| `backend/sheets.py` | `get_kindergartens()` の読み込みに `address`・`area` を追加 |
| `backend/sheets.py` | `update_kindergarten_master()` の mapping・auto-create に `address`・`area` を追加 |
| 管理画面 幼稚園マスター編集UI | 住所入力欄を追加し、入力時にエリアを自動抽出・セット |

## ① 注文一覧ビュー

### 目的
指定した日付の全幼稚園の注文をテーブルで一覧表示。
電話対応時に「○○幼稚園は今日何食？」をすぐ確認できる。

### UI構成

```
[ 日付ピッカー: 2026/03/27 ] [ 読み込む ]

エリア | 園名         | 種別  | 園児  | アレ | 先生 | 合計 | メモ
-------|------------|-------|------|------|------|------|-----
北区   | ひまわり幼稚園 | 通常  |  20  |  2  |  3  |  25  |
北区   | さくら保育園  | カレー |  15  |  1  |  2  |  18  |
南区   | たんぽぽ幼稚園 | 通常  |  30  |  3  |  4  |  37  |
```

- **クラスあり園**：クラスごとに行を展開 or クラス合計を1行で表示（折りたたみ）
- **クラスなし園**：1行で表示
- エリアでグループ化（エリア未設定は「エリア未設定」グループ）
- 注文なしの園はグレーアウト or 非表示（切り替えボタン）
- ページ内検索（園名フィルタ）

### データ取得
- 新バックエンドエンドポイント `GET /admin/daily-orders/{date}` を使用
- `date` は `YYYY-MM-DD` 形式

## ② 配送リストビュー

### 目的
同日の配送計画立案用。エリア別・配送順を意識したリスト。

### UI構成

```
配送リスト：2026年3月27日

[ エリアでグループ表示 ▼ ]  [ 印刷 ]

── 北区 ────────────────────────────────
  1. ひまわり幼稚園   計25食（園児22 / 先生3）  通常
  2. さくら保育園     計18食（園児16 / 先生2）  カレー

── 南区 ────────────────────────────────
  3. たんぽぽ幼稚園   計37食（園児33 / 先生4）  通常

合計：80食
```

- エリアごとにブロック分け
- 各園の総食数（園児+アレルギー+先生）を強調表示
- 給食タイプ（通常/カレー等）を表示
- 印刷ボタンで印刷用レイアウトに切り替え
- 注文なし園は非表示

## 新バックエンドAPI

### `GET /admin/daily-orders/{date}`

指定日の全幼稚園の注文を返す。

**レスポンス例:**
```json
{
  "date": "2026-03-27",
  "kindergartens": [
    {
      "kindergarten_id": "K001",
      "name": "ひまわり幼稚園",
      "area": "北区",
      "has_orders": true,
      "orders": [
        {
          "class_name": "ひよこ組",
          "meal_type": "通常",
          "student_count": 10,
          "allergy_count": 1,
          "teacher_count": 2,
          "memo": ""
        }
      ],
      "totals": {
        "student": 20,
        "allergy": 2,
        "teacher": 3,
        "grand_total": 25
      }
    }
  ],
  "grand_total": 80
}
```

## 管理画面ナビゲーション変更

現状の `activeSection` に `'daily'` を追加：

```typescript
type ActiveSection = 'menu' | 'kindergarten' | 'orders' | 'daily' | null
```

ダッシュボードに新カードを追加：

```
[ 今日の注文 ]
注文確認・配送計画
```

## 非機能要件

- レスポンス速度：読み込みボタン押下後3秒以内に表示
- 印刷：配送リストはA4縦で印刷できること
- モバイル対応：スマートフォンからも確認できること（最低限の縦スクロール対応）
